Media Access Control

Data Link Frame

The data link layer provides addressing that is used in transporting a frame across a shared local media. Device addresses at this layer are referred to as physical addresses. Data link layer addressing is contained within the frame header and specifies the frame destination node on the local network. The frame header may also contain the source address of the frame.

Physical addresses are not hierarchical in nature. That means that the physical address does not indicate on what network the device is located, but rather is a unique address among all devices, everywhere. If the device is moved to another network or subnet, it will still function with the same Layer 2 physical address.

By having an address that is device specific and non-hierarchical, the address cannot be used to locate a device. There are simply too many devices connected to the Internet! The address, however, can be used to locate a device within a limited area, with a limited number of devices to choose from. For this reason, the data link layer address is only used for local delivery. Addresses at this layer have no meaning beyond the local network.

If the data packet must pass onto another network segment, the intermediate device - a router - must accept the frame, de-encapsulate the frame, examine the hierarchical Layer 3 address to determine where to locate the destination device, and then create a new frame for the packet, and send it onto the new segment toward its final destination. The new frame will use source and destination physical addressing as necessary to transport the packet across the new media.
